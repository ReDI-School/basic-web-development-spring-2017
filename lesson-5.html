Pseudo classes

You may change property values of elements, if you hover them

Example:

.link { color: blue; } .link:hover { color: red; }

Pseudo classes

You may change property values of elements, if you focus/click them

Example:

.link { color: blue; } .link:active { color: green; }

Pseudo classes

You may change property values of elements, if they are in the browser history

Example:

.link { color: blue; text-decoration: underline; } .link:visited { color: blue; text-decoration: none; }

Block elements vs. inline elements

Some elements behave like a block.
They form an invisble box around their children and are 100% in width.

The next element after a block element, will go into the next line

You know a lot of block elements already:

  • main
  • p
  • section
  • li

What else are block elements?

Block elements vs. inline elements

Some elements behave like a character in a text. They are called inline elements.

The next inline element after an inline element, will be next to the inline element.

You know a some inline elements already:

  • em
  • strong
  • span

What else are inline elements?

CSS display property

You can change between block and inline element behaviour using the property display.

CSS

.listitem { display: inline; }

HTML

<ul> <li class="listitem">Item 1</li> <li class="listitem">Item 2</li> <li class="listitem">Item 3</li> </ul>

What happens? Test it in your browser!

CSS display property

More values for display:

inline elements are inline, they don't have a width
inline-blockelements are inline, but they have a width
blockelements are beneath each other and have a width
noneelements are not rendered (are invisible and do not take up space)

Debugging

What is debugging?

It helps you find errors more easily. For example missing selectors or wrong property values.

  • 1. Open a page on Firefox
  • 2. Right click on the page
  • 3. Select Inspect element

The debugger will be opened.
You may navigate through the elements and see their CSS properties

Everything you change in the debugger, will not be saved!
